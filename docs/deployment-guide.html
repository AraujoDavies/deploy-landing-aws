1Prerequisites

Before running any Terraform command, make sure the following tools and accesses are in place.

Required tools

  • Terraform ≥ 1.6 — terraform -version
  • AWS CLI v2 — aws --version
  • AWS credentials configured — aws configure or environment variables

Required AWS permissions

The IAM user or role running Terraform needs permissions for: S3, CloudFront, ACM (us-east-1), Route 53, and KMS (if re-enabled).

S3 remote state bucket (backend.tf)

Before the first terraform init, edit backend.tf and replace the placeholders:

terraform {
  backend "s3" {
    bucket  = "YOUR-TFSTATE-BUCKET-NAME"   # must exist beforehand
    key     = "static-website/terraform.tfstate"
    region  = "us-east-1"
    encrypt = true
  }
}
⚠ Important
Create the S3 state bucket manually (or with a bootstrap script) before running terraform init. Terraform cannot create its own backend. State locking (DynamoDB) is not used — avoid running concurrent terraform apply from multiple machines.

3Dev Deployment

Dev uses the default *.cloudfront.net certificate — no custom domain needed. It is the fastest way to get the site live.

Step 1 — Initialize Terraform

terraform init

Downloads the AWS provider and configures the S3 backend. Run this once per machine.

Step 2 — Plan & Apply

# Preview what will be created (no changes made)
terraform plan -var-file="envs/dev.tfvars"

# Create all resources
terraform apply -var-file="envs/dev.tfvars"

Terraform will create: 1 S3 bucket (content), 1 CloudFront distribution. Takes ~3 minutes.

Step 3 — Upload HTML files

Bash / macOS / Linux

# Get the bucket name from outputs
BUCKET=$(terraform output -raw content_bucket_name)

# Upload both pages
aws s3 cp index.html s3://$BUCKET/index.html
aws s3 cp error.html s3://$BUCKET/error.html

PowerShell (Windows)

# Get the bucket name from outputs
$BUCKET = terraform output -raw content_bucket_name

# Upload both pages
aws s3 cp index.html "s3://$BUCKET/index.html"
aws s3 cp error.html "s3://$BUCKET/error.html"

Step 4 — Open the website

terraform output website_url
# → https://d1234abcd.cloudfront.net
✓ Dev is live
The site is accessible at the CloudFront URL. No domain registrar changes needed.

4Prod Deployment (Custom Domain)

Domain prerequisites

  1. Register your domain Buy a domain at any registrar (GoDaddy, Namecheap, Registro.br, etc.) or directly in AWS Route 53 Registrar.
  2. Create a Route 53 Hosted Zone In the AWS Console → Route 53 → Hosted Zones → Create hosted zone.
    Enter your domain name (e.g. ataqtech.com) and leave type as Public.
  3. Note the 4 NS records After creation, AWS shows 4 nameserver values. You will need these in the next section.
  4. Update nameservers at your registrar See Section 5 for the step-by-step registrar guide.

Configure prod.tfvars

Open envs/prod.tfvars and replace the domain placeholder:

domain_name = "ataqtech.com"   # ← your real domain here

Plan & Apply

terraform plan  -var-file="envs/prod.tfvars"
terraform apply -var-file="envs/prod.tfvars"

Terraform will additionally create: ACM certificate (us-east-1), Route 53 DNS validation CNAMEs, Route 53 A + AAAA alias records. Takes ~10–15 minutes (ACM validation waits for DNS to propagate).

ℹ ACM validation wait
Terraform blocks at the aws_acm_certificate_validation resource until AWS confirms the DNS CNAME is reachable. If it times out, nameserver propagation is not complete yet — wait and re-run terraform apply.

Upload HTML files

Bash / macOS / Linux

BUCKET=$(terraform output -raw content_bucket_name)

aws s3 cp index.html s3://$BUCKET/index.html
aws s3 cp error.html s3://$BUCKET/error.html

PowerShell (Windows)

$BUCKET = terraform output -raw content_bucket_name

aws s3 cp index.html "s3://$BUCKET/index.html"
aws s3 cp error.html "s3://$BUCKET/error.html"

Check outputs

After a successful apply, terraform output prints:

5Configuring Nameservers at Your Registrar

This is a one-time manual step done outside of Terraform — in your domain registrar's control panel.

Where to find the nameserver values

After terraform apply, run:

terraform output route53_nameservers

Or find them in the AWS Console → Route 53 → Hosted Zones → your domain → NS record.

Example nameservers (yours will be different):

ns-8.awsdns-47.net
ns-1.awsdns-60.org
ns-1.awsdns-57.co.uk
ns-1.awsdns-19.com
⚠ Remove trailing dots
Some registrars don't accept the trailing period (.). Remove it if the form rejects the values.

Step-by-step per registrar

RegistrarPath to Nameserver Settings
GoDaddyMy Products → DNS → Nameservers → Change → Enter my own nameservers
NamecheapDomain List → Manage → Nameservers → Custom DNS
Registro.brDomínios → seu domínio → Servidores DNS → Configurar
Cloudflare RegistrarDomain Registration → your domain → Nameservers
Google / SquarespaceDNS → Custom name servers → Use custom name servers
AWS Route 53 RegistrarNot needed — hosted zone is already authoritative

Verify propagation

Propagation takes 5 minutes to 48 hours. Check with a public DNS tool:

  • dnschecker.org — search your domain, select record type NS
  • whatsmydns.net — shows results per region globally

When all regions show your Route 53 NS values, ACM validation will complete automatically on the next terraform apply (or it already completed if propagation happened during apply).

7Cache Invalidation

After uploading new HTML files, CloudFront may serve the old cached version for up to 24 hours (the default TTL). Run a cache invalidation to force immediate refresh:

Bash / macOS / Linux

DIST_ID=$(terraform output -raw cloudfront_distribution_id)

# Invalidate all files
aws cloudfront create-invalidation \
  --distribution-id $DIST_ID \
  --paths "/*"

# Invalidate a specific file only
aws cloudfront create-invalidation \
  --distribution-id $DIST_ID \
  --paths "/index.html"

PowerShell (Windows)

$DIST_ID = terraform output -raw cloudfront_distribution_id

# Invalidate all files
aws cloudfront create-invalidation `
  --distribution-id $DIST_ID `
  --paths "/*"

# Invalidate a specific file only
aws cloudfront create-invalidation `
  --distribution-id $DIST_ID `
  --paths "/index.html"
ℹ Cost note
The first 1,000 invalidation paths per month are free. Each additional 1,000 paths costs $0.005. For dev, set cloudfront_default_ttl = 0 in dev.tfvars to disable caching entirely.

8Destroying the Infrastructure

⛔ Warning — irreversible
This permanently deletes the S3 content bucket and its contents, the CloudFront distribution, ACM certificate, and Route 53 records. In prod, force_destroy = false means S3 will refuse deletion if the bucket is not empty — empty it first.

Dev

terraform destroy -var-file="envs/dev.tfvars"

Prod

Bash / macOS / Linux

# Empty the bucket first (prod has force_destroy = false)
BUCKET=$(terraform output -raw content_bucket_name)
aws s3 rm s3://$BUCKET --recursive

# Then destroy
terraform destroy -var-file="envs/prod.tfvars"

PowerShell (Windows)

# Empty the bucket first (prod has force_destroy = false)
$BUCKET = terraform output -raw content_bucket_name
aws s3 rm "s3://$BUCKET" --recursive

# Then destroy
terraform destroy -var-file="envs/prod.tfvars"
